How it works

Three steps. Zero learning curve.

No collections. No environments. No workspaces. Just the parts of API testing you actually use.

01

Add your API keys

Paste your keys into the vault — Stripe, OpenAI, Twilio, whatever you use. Reference them with a variable name across every template. One entry, everywhere.

02

Build a template

Define your HTTP request and mark dynamic parts with {{placeholders}}. DevBook generates a fillable form. No raw JSON editing, no config files.

03

Run and reuse

Fill in the blanks, hit send, see your response instantly. Every template is saved and searchable. Build a library of the API calls your workflow depends on.

Security

How we protect your API keys

You're trusting us with credentials — here's exactly what we do with them.

🔐

AES-256-GCM encryption

Every API key stored in the vault is encrypted at rest using AES-256-GCM — the same standard used by banks and cloud providers. Keys are decrypted only when you use them.

🔒

TLS everywhere

All connections between your browser and DevBook use TLS encryption. Your requests, responses, and credentials are never transmitted in plaintext.

🚫

No payload logging

We never log your request bodies or API responses on our servers. Your data passes through to execute the request and is not stored or analyzed.
